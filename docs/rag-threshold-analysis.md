# RAG Similarity Threshold 분석

## 문제 정의

`backend/main.py` 의 `_search_rag()` 함수는 벡터 검색 결과를 `score > 0.3` 으로 필터링하고 있었다.

```python
if score > 0.3   # ← 근거 없이 하드코딩
```

이 임계값이 실제로 **관련 있는 문서**만 통과시키는지, 아니면 **관련 없는 문서도 섞여 들어오는지** 검증이 필요했다.

## 실험

### 방법

- 임베딩: `BAAI/bge-m3` (정규화 코사인 유사도, score 범위 대략 0~1)
- 벡터스토어: `ChromaDB` (9,071 chunks)
- 테스트 쿼리 10개:
  - **관련 있는 쿼리 9개**: 회사별 실제 면접 주제 (`토스 결제 시스템 아키텍처`, `Kafka 기반 데이터 파이프라인`, `MySQL 샤딩 전략` 등)
  - **음성 대조군(negative control) 1개**: `블록체인 NFT 투자 전략` — 면접 준비 맥락과 무관한 투자성 주제
- 각 쿼리마다 top_k=10, total 100개 결과 수집
- 재현: `python scripts/analyze_threshold.py`

### 결과 — 전체 score 분포

| 구간 | 개수 | 비율 |
|------|------|------|
| 0.55~ | 1  | 1%  |
| 0.50~0.55 | 7  | 7%  |
| 0.45~0.50 | 26 | 26% |
| 0.40~0.45 | 34 | 34% |
| 0.35~0.40 | 24 | 24% |
| 0.30~0.35 | 4  | 4%  |
| 0.25~0.30 | 4  | 4%  |

- Max: 0.5282, Median: 0.4029, Min: 0.2572
- **0.30 이상**: 92% 통과
- **0.40 이상**: 52% 통과

### 핵심 발견 — 음성 대조군

"블록체인 NFT 투자 전략" 쿼리에 대한 결과:

| rank | score | title | 관련성 판단 |
|------|-------|-------|------------|
| 1 | **0.3647** | "Klip App2App API를 활용한 데모 앱 개발기" (카카오) | 무관 — NFT 단어 매칭만 됨 |
| 2 | 0.3296 | "Klaytn, Caver, KAS가 만들어가는 BApp" (카카오) | 무관 |
| 3 | 0.2916 | "비트코인과 블록체인 기술" (네이버) | 무관 |
| 4~10 | 0.25~0.29 | 블록체인/WebAssembly/RL 관련 | 무관 |

**음성 대조군의 max score = 0.3647**

즉 면접 맥락과 전혀 관련 없는 쿼리도 score 0.36 까지는 나온다. 현재 임계값 0.30이면 이런 무관한 결과가 **3개**(rank 1~3 중 2개) 통과되어 LLM 프롬프트에 들어간다.

→ **0.30은 과소 필터링(under-filtering)**

## 결정

**임계값을 0.30 → 0.40 으로 상향.**

### 근거

| 기준 | 0.30 (Before) | 0.40 (After) |
|------|--------------|--------------|
| 음성 대조군 통과 | 1개 (0.3647) | 0개 |
| 관련 쿼리 상위권 유지 | 92% | 52% |
| 평균 관련도 | 낮음 (섞임) | 높음 (상위권만) |

### Trade-off

- **장점**: 무관 문서 배제로 정밀도(precision) 상승 → LLM 환각(hallucination) 감소, 사용자가 보는 "참고 출처"의 신뢰도 상승
- **단점**: 재현율(recall) 감소 — 관련 있지만 score 0.3~0.4 구간의 문서는 배제됨
- **보완**: Multi-Query Rewriting(Task 3)으로 쿼리를 3개 관점으로 확장해 검색함으로써 재현율 감소분을 상쇄

## 결과

- `backend/main.py` 에 `RAG_SIMILARITY_THRESHOLD` 환경변수 도입 (기본값 0.4)
- `.env.example` 에 기본값 명시
- 향후 데이터셋 특성이 바뀌면 동일 스크립트(`scripts/analyze_threshold.py`)로 재측정 후 조정 가능

```bash
# ChromaDB 준비 후
python scripts/analyze_threshold.py
```
